## 2024-06-25 - [Pre-allocate array bounds in high-frequency candidate generation]
 **Learning:** In `generateAllCandidates` within `public/app.js`, dynamically resizing an array up to ~6000 items inside hot loops via `.push()` forces V8 to repeatedly allocate memory and run garbage collection (`GrowFastSmiOrObjectElements`).
 **Action:** Instead of dynamically pushing, always pre-calculate a theoretical maximum upper bound (e.g., `numWorkdays * allowance`), pre-allocate the array via `new Array(maxBounds)`, assign sequentially using an incremented counter (`candCount`), and truncate the array property `.length = candCount` at the end to return exactly the generated set. This completely eliminates resizing overhead and speeds up the entire optimal planning chain significantly.
