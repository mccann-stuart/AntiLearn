## 2025-04-03 - Optimizing V8 Array Allocations in Candidate Generation
**Learning:** Dynamic array resizing (using `.push()`) inside the hot `generateAllCandidates` loop triggers expensive V8 `GrowFastSmiOrObjectElements` operations and significant garbage collection thrashing. While reusing a global module-level array fixes the GC pressure, it introduces dangerous global mutable state that breaks concurrent safety and fails code review.
**Action:** Always pre-calculate the exact required array size (e.g., predicting total candidates via `Math.min(allowance, numWorkdays - k)`) and allocate it precisely (`new Array(totalCandidates)`). This eliminates dynamic resizing overhead while keeping the state safely scoped to the function invocation.
