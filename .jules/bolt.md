## 2024-05-02 - FindOptimalPlan Memory Optimization
 **Learning:** The `findOptimalPlan` function does a very expensive, complex DP calculation, and was repeatedly called whenever the year changed (once to generate the current year plan, and again to compare it with the previous year via `getYearComparison`, generating both plans from scratch).
 **Action:** Introduce an `optimalPlanCache` inside `public/app.js` with comprehensive cache keys (`year`, `allowance`, `currentRegion`, `currentWeekendPattern`, `customCount`, `datasetKey`), and tie its invalidation to `invalidateInsightCaches()` to prevent duplicate computational load for the exact same constraints. This reduces repeated `findOptimalPlan` generation time by 99%.
